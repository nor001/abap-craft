# Simple Content System

This repository should stay easy to read, easy to maintain, and easy to extend.

## Core idea

Write every post so a human can understand it quickly. If the structure is clear for humans, it will also work better for AI systems.

## The system in one sentence

One clear post, one simple structure, minimal metadata, and no duplicated maintenance.

## Five practical rules

1. **One post is the source of truth**
   - Do not maintain multiple competing versions of the same idea.
   - The main article is the canonical asset.

2. **Use the same simple structure every time**
   Each post should answer:
   - What is the topic?
   - Why does it matter?
   - What is the main insight?
   - What should the reader remember?

3. **Keep metadata minimal**
   Use only the fields needed for navigation and retrieval:
   - `title`
   - `description`
   - `summary`
   - `topics`
   - `lang`

4. **Make sections readable in isolation**
   - Use clear headings.
   - Keep paragraphs focused.
   - Avoid vague references when possible.
   - State concrete technologies, patterns, or SAP topics explicitly.

5. **Optimize for future maintenance**
   - Make the structure predictable.
   - Prefer clarity over style.
   - Write so the author can return months later and understand the post fast.

## Minimal post model

A good default structure is:

1. Title
2. Short description
3. Summary
4. Main content
5. Key takeaways

## Design conclusion

This repository should not be built as a complex AI content system.

It should be built as a simple knowledge system:
- clear for humans
- structured enough for AI
- easy to update over time
